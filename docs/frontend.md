# Frontend — CoworkWise

**Технологии:** Next.js 15 (App Router), TypeScript, Tailwind CSS v4, Recharts, Leaflet, Zustand, shadcn/ui, Docker

Фронтенд — это визуальная часть платформы, то что видит пользователь в браузере. Запускается в Docker-контейнере на порту 3000, общается с бэкендом через REST API.

---

## Страницы и что они делают

### `/login` и `/register`
Вход и регистрация. Отправляют запросы на `POST /auth/login` и `POST /auth/register`.
После входа JWT-токен сохраняется в localStorage и используется для всех последующих запросов.

### `/map` — Карта (главная рабочая страница)
- Отображает тепловую карту пешеходного трафика (Leaflet + данные из `/telecom/grids/with_activity`)
- Показывает слои: коворкинги, транспорт, деловые районы
- Два режима: **Explore** (клик → аналитика точки) и **Compare** (выбор 2+ точек для сравнения)
- Панель фильтров: район, радиус поиска, время суток
- Зелёные ML-метки — топ-рекомендации модели на карте

Компоненты: `map-view.tsx`, `map-filters.tsx`

### `/analysis` — Анализ (4 вкладки)

**Сравнение (`compare-view.tsx`)**
- Три источника данных: сессия карты / сохранённые точки / кандидаты модели
- Таблица с колонками: трафик, коворкинги, конкуренция, аренда, итоговый рейтинг
- Формула рейтинга: `трафик − конкуренция × 3500 − трафик × 0.25 × (аренда − min) / (max − min)`
- Три сводные карточки: лучший сейчас / лучший баланс / высокий риск
- Гистограмма трафика по локациям (Recharts)

**Рекомендации (`recommendations-view.tsx`)**
- Список до 15 лучших локаций по версии ML-сервиса
- Каждая карточка: оценка модели (0–100), преимущества, трафик, аренда, рост
- Действия: смотреть на карте / сохранить / скопировать резюме

**Сохранённые (`saved-places-view.tsx`)**
- Хранятся в localStorage браузера (не в БД — не привязаны к аккаунту)
- Фильтрация: все / из рекомендаций / с карты
- Переход к полному сравнению

**Прогноз (`forecast-view.tsx`)**
- Линейный график трафика по месяцам (реальные данные + прогноз +10%)
- Таблица прогноза по районам: тренд роста, инфраструктура, конкуренция, итоговая оценка

### `/admin` — Панель администратора
Доступна только пользователям с ролью `admin`.

- **Пользователи** — список, добавление, назначение/снятие роли, удаление
- **Данные** — загрузка `.xlsx` телеком-файла (месяц + файл → `POST /telecom/upload`), включение/отключение слоёв карты
- **Настройки** — название приложения, часовой пояс, язык по умолчанию, таймаут сессии, доступ к API

### `/profile` — Профиль
Изменение имени, email и пароля.

## Глобальное состояние (Zustand stores)

| Store | Что хранит |
|---|---|
| `useMapStore` | Выбранный район, радиус, время суток, режим карты, точки сессии сравнения |
| `useSavedPlacesStore` | Сохранённые точки (persist в localStorage) |
| `useLangStore` | Выбранный язык (persist в localStorage) |

## Интернационализация (i18n)

Три языка: **English / Русский / Қазақша**

Все строки интерфейса хранятся в `src/lib/translations/` (en.ts, ru.ts, kk.ts).
Тип `Translations` выводится из английского файла — TypeScript ошибается если в ru/kk не хватает ключа.
Хук `useT()` возвращает нужный словарь по текущему языку.

Язык и тема (тёмная/светлая) доступны на всех страницах включая логин и регистрацию.
