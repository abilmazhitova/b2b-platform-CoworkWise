# ML-слой — CoworkWise

**Важно:** в платформе нет обученной нейросети или классической ML-модели. Используется **rule-based scoring** — взвешенная формула по нескольким факторам. Это осознанный выбор: для обучения настоящей модели нужны исторические данные (открылся коворкинг → выжил/закрылся), которых пока нет.

---

## Из чего состоит ML-слой

### Единственный рабочий файл — `b2b-backend/app/services/ml_analysis_service.py`

Вся ML-логика живёт здесь. Запускается в реальном времени при каждом запросе пользователя, работает с данными из PostgreSQL.

`b2b-ml_analysis/` — это **архивный исследовательский проект**, где логика разрабатывалась и тестировалась на CSV-файлах. В продакшне он не используется. Из него были взяты только два статичных файла данных, скопированных в бэкенд:
- `infra_points.csv` (OSM-инфраструктура) → `b2b-backend/data/`
- `districts.geojson` (границы районов) → `b2b-backend/data/`

Единственный скрипт из `b2b-ml_analysis` который может понадобиться в будущем — `data/extract_osm_infra.py` для обновления OSM-данных.

## Как работает scoring (главная логика)

### Шаг 1 — Данные из БД
Для каждой телеком-ячейки берётся сумма уникальных пользователей с фильтром по времени суток и дню недели (если выбраны).

### Шаг 2 — Геопривязка
Каждая ячейка через `geopandas` сопоставляется с районом Алматы (`districts.geojson`).

### Шаг 3 — Инфраструктура из OSM
В радиусе 700м от каждой ячейки считается количество объектов из `infra_points.csv`:
- `infra_score` — кафе, офисы, университеты, торговые центры, спорт
- `accessibility` — остановки автобусов и метро
- `competition` — уже существующие коворкинги

### Шаг 4 — Аренда
Из `rent_index.json` берётся медиана аренды (тг/м²/мес) для района точки.
Источник: 455 объявлений Krisha.kz, скрапер в `scraper/krisha_scraper.py`.

### Шаг 5 — Нормализация
Все метрики приводятся в диапазон 0–1 через `MinMaxScaler` из sklearn.
Это не обучение — просто масштабирование: `(x − min) / (max − min)`.

### Шаг 6 — Итоговый балл

```
potential_score =
    0.30 × трафик_норм
  + 0.23 × инфраструктура_норм
  + 0.17 × транспорт_норм
  − 0.15 × конкуренция_норм
  − 0.15 × аренда_норм
```

Высокий трафик и развитая инфраструктура повышают балл.
Много коворкингов рядом или дорогая аренда — снижают.

### Шаг 7 — Описание преимуществ
Текстовые метки формируются автоматически по порогам:
- `avg_density_scaled > 0.6` → "High people density"
- `rent_m2_scaled < 0.3` → "Affordable rent"
- `competition_scaled > 0.6` → "High competition area"
и т.д.

## Прогноз по районам (`get_forecast_districts_ml`)

Отдельная функция для вкладки Прогноз:
- `growth_trend` — ранг ячейки по трафику + небольшой шум (seed=42)
- Агрегация по районам: среднее по тренду, инфраструктуре, конкуренции
- Формула: `0.5 × тренд + 0.3 × инфра − 0.2 × конкуренция`
- Категории: High Growth / Moderate Growth / Low Growth

## Данные которые используются

| Источник | Что даёт | Как обновлять |
|---|---|---|
| Телеком-оператор (.xlsx) | Количество уникальных пользователей по ячейкам, час, день | Загрузка через Admin → Данные |
| OpenStreetMap (Overpass API) | Инфраструктура: кафе, транспорт, коворкинги | Вручную, скрипт `extract_osm_infra.py` |
| Krisha.kz (парсинг) | Медиана аренды коммерческой недвижимости по районам | Вручную раз в месяц, скрипт `scraper/krisha_scraper.py` |
| districts.geojson | Границы 8 районов Алматы | Статично, из OpenStreetMap |

## Структура проекта

```
b2b-backend/app/services/
└── ml_analysis_service.py     — ВСЯ ML-логика (работает с PostgreSQL в реальном времени)

b2b-backend/data/
├── districts.geojson          — границы 8 районов Алматы
├── infra_points.csv           — OSM-инфраструктура (кафе, транспорт, коворкинги)
└── rent_index.json            — медиана аренды по районам (Krisha.kz)

scraper/
└── krisha_scraper.py          — парсинг аренды с Krisha.kz (запускать раз в месяц)

b2b-ml_analysis/               — АРХИВ, в продакшне не используется
└── ml/data/extract_osm_infra.py  — единственный скрипт который может понадобиться:
                                     обновить infra_points.csv из OpenStreetMap
```
