# CoworkWise — Руководство пользователя

CoworkWise — аналитическая платформа для выбора оптимального расположения коворкинг-центра в Алматы. Платформа использует данные телеком-операторов, OSM-инфраструктуру и плотность существующих коворкингов для формирования рекомендаций.

## 1. Вход и регистрация

### Вход

1. Откройте платформу в браузере.
2. Введите **Email** и **Пароль**.
3. Нажмите **Войти**.

После успешного входа вы попадёте на страницу **Карта**.

### Регистрация

1. На странице входа нажмите **Регистрация**.
2. Заполните поля:
   - **Полное имя** — необязательно
   - **Email** — будет использоваться для входа
   - **Пароль** — минимум 6 символов
   - **Подтвердите пароль**
3. Нажмите **Зарегистрироваться**.

> Если вы забыли пароль — нажмите «Забыли пароль?» и обратитесь к администратору платформы.

## 2. Карта

Карта — основной рабочий инструмент. Здесь отображается тепловая карта пешеходного трафика, расположение коворкингов и рекомендации модели.

### Режимы карты

| Режим | Описание |
|---|---|
| **Explore** | Исследование карты, клик по точке показывает её характеристики |
| **Compare** | Выбор точек для сравнения: кликните 2+ точки, затем нажмите **Full Compare** |

Переключатель режимов находится в левом верхнем углу карты.

### Панель фильтров (слева)

**Район** — ограничить анализ одним из районов Алматы:
- Алмалы, Ауэзов, Бостандык, Медеу, Турксиб, Жетысу, Алатау, Наурызбай

**Радиус поиска** — от 25 м до 2 км. Влияет на расчёт конкуренции и инфраструктуры вокруг выбранной точки.

**Время суток** — фильтрует данные по часам активности:
- Утро (6–12), День (12–18), Вечер (18–24), Весь день

**ML рекомендации на карте** — зелёные пронумерованные метки показывают топ-локации по версии модели. Цифра на метке — это ранг рекомендации.

### Статистика под фильтрами

После выбора района и времени в нижней части панели автоматически обновляется статистика:

| Показатель | Что означает |
|---|---|
| Ячеек сетки | Количество телеком-ячеек в выбранной зоне |
| С активностью | Ячейки, где зафиксированы пользователи |
| Ср. активность | Средний уровень трафика по ячейкам |
| Горячие зоны | Ячейки с активностью ≥ 50% от максимума |

### Клик по точке на карте

В режиме **Explore** клик открывает боковую панель с характеристиками точки:
- Район
- Плотность трафика (density)
- Уровень конкуренции (количество коворкингов в радиусе)
- Инфраструктура рядом (кафе, транспорт и т.д.)

Здесь же можно **Сохранить** точку для последующего сравнения.

### Режим Compare

1. Переключитесь в режим **Compare**.
2. Кликните на карте минимум **2 точки** — они добавятся в список слева.
3. Нажмите **Full Compare** — вы перейдёте на вкладку Анализ → Сравнение с данными по этим точкам.

## 3. Анализ

Раздел состоит из четырёх вкладок: **Сравнение**, **Рекомендации**, **Сохранённые**, **Прогноз**.

### 3.1 Сравнение

Позволяет сравнить несколько локаций по ключевым показателям.

#### Источники данных для сравнения

В верхней части страницы три кнопки-переключателя:

| Кнопка | Что показывает |
|---|---|
| **Сессия карты (N)** | Точки, которые вы кликнули на карте в режиме Compare в эту сессию |
| **Мои точки (N)** | Локации, которые вы сохранили через кнопку «Сохранить» |
| **Кандидаты модели (N)** | Топ-локации, отранжированные моделью автоматически |

> Кнопка доступна только если выбранных точек **2 или больше**.

#### Сводные карточки

Три карточки показывают итоговые выводы по текущему набору данных:

- **Лучший сейчас** — локация с наибольшим спросом за вычетом конкуренции
- **Лучший баланс** — хороший трафик при умеренной конкуренции
- **Высокий риск конкуренции** — локация с наибольшим количеством коворкингов рядом

#### Таблица локаций

Показывает все точки текущего набора, отсортированные по рейтингу:

| Колонка | Описание |
|---|---|
| # | Ранг (1 = лучший) |
| Район / Ячейка | Название района или ID телеком-ячейки |
| Трафик | Расчётный пешеходный трафик |
| Коворкинги | Количество коворкингов в радиусе |
| Конкуренция | Оценка конкуренции (0–10) |
| Рейтинг | Итоговый балл |

#### График трафика

Столбчатая диаграмма с трафиком по каждой локации для быстрого визуального сравнения.

---

### 3.2 Рекомендации

Список лучших локаций по версии модели — до 15 предложений, отранжированных по оценке.

Каждая карточка содержит:
- **Ранг и район** — порядковый номер и название района
- **Оценка модели** — от 0 до 100
- **Ключевые преимущества** — текстовые причины, почему место перспективно
- **Расчётный трафик** — проксирует пешеходный поток
- **Рост (оценка)** — прогнозируемая динамика

#### Действия с рекомендацией

| Кнопка | Действие |
|---|---|
| **Смотреть на карте** | Переход на карту с центрированием на этой локации |
| **Сохранить** | Добавить в список «Сохранённые» для дальнейшего сравнения |
| **Скопировать** | Копировать краткое резюме в буфер обмена (для вставки в письмо или отчёт) |

> Фильтры района и времени суток из Карты влияют и на список рекомендаций — при выборе конкретного района показываются только рекомендации из этого района.

---

### 3.3 Сохранённые

Список всех локаций, которые вы сохранили — как из Рекомендаций, так и кликом на Карте.

> **Важно:** данные хранятся в локальном хранилище браузера. Они не привязаны к аккаунту и не синхронизируются между устройствами.

#### Фильтры

- **Все** — полный список
- **Рекомендации** — только сохранённые из раздела Рекомендации
- **Мои точки** — только точки, сохранённые с карты вручную

#### Сводные карточки

При наличии 2+ сохранённых точек платформа выделяет:
- **Наибольшая активность** — точка с максимальным трафиком
- **Наименьшая конкуренция** — точка с наименьшим числом коворкингов рядом

#### Таблица

Показывает все сохранённые точки. По каждой можно:
- Нажать **иконку карты** — перейти на карту к этой точке
- Нажать **иконку удаления** — убрать точку из списка

**Открыть полное сравнение** — переходит на вкладку Сравнение с источником «Мои точки».

**Очистить всё** — удалить все сохранённые точки.

---

### 3.4 Прогноз

Показывает динамику спроса на основе загруженных телеком-данных.

#### Прогноз роста

Карточка с общим трендом: процентное изменение трафика от первого до последнего месяца в данных.

#### График по месяцам

Линейный график с несколькими рядами:
- **Actual** (синий) — реальные данные из загруженных файлов
- **Predicted** (синий пунктир) — прогноз модели
- **Lower / Upper band** (серые) — погрешность прогноза

#### Прогноз роста по районам

Таблица с оценкой каждого района по четырём показателям: тренд роста, инфраструктура, конкуренция, итоговый балл.

> Если данных нет — платформа отобразит соответствующее сообщение. Данные загружаются администратором через раздел Данные в панели администратора.

## 4. Профиль

Доступен через кнопку **Профиль** в навигации.

### Личная информация

- **Полное имя** — отображается в интерфейсе
- **Email** — используется для входа

Нажмите **Сохранить** для применения изменений, **Отмена** — для сброса.

### Изменить пароль

1. Введите **Текущий пароль**
2. Введите **Новый пароль** (минимум 6 символов)
3. Подтвердите новый пароль
4. Нажмите **Обновить пароль**

### Выход

Кнопка **Выйти** находится на карточке профиля (вверху страницы) и в правой части навигационной панели.

## 5. Панель администратора

Доступна только пользователям с ролью **Администратор**.

### Вкладка: Пользователи

Список всех зарегистрированных пользователей платформы.

**Добавить пользователя:**
1. Нажмите **Добавить пользователя**
2. Заполните Email, Имя (необязательно), Пароль
3. Нажмите **Создать**

**Действия над пользователем** (меню ⋮):
- **Сделать администратором** / **Снять права** — переключить роль
- **Удалить пользователя** — необратимое действие

> Вы не можете изменить или удалить свой собственный аккаунт через этот интерфейс.

### Вкладка: Данные

**Слои данных** — список слоёв карты (Footfall Heatmap, Coworking Locations, Public Transport, Business Districts). Каждый слой можно включить или отключить переключателем.

**Загрузить телеком данные:**

Это основная операция для обновления аналитики платформы.

Требования к файлу:
- Формат: **Excel (.xlsx)**
- Обязательные колонки:
  - `ZID_NUMBER` — ID телеком-ячейки
  - `lat`, `lon` (или аналогичные) — координаты
  - `WEEK_DAY_IND` — день недели (1=будни, 2=выходные)
  - `TIME_HOUR` — час (0–23)
  - `NUM_OF_UNIQ_USERS` — количество уникальных пользователей

Порядок загрузки:
1. Укажите **Месяц** в формате `MM.YYYY` (например, `03.2023`)
2. Выберите файл `.xlsx`
3. Нажмите **Загрузить**

После успешной загрузки данные появятся в графиках Прогноза и обновят тепловую карту.

### Вкладка: Настройки

- **Название приложения** — отображается в заголовке
- **Часовой пояс** — Asia/Almaty (GMT+6) по умолчанию
- **Язык по умолчанию** — язык интерфейса
- **Таймаут сессии** — автоматический выход после бездействия (15 мин / 30 мин / 1 час / никогда)
- **Доступ к API** — разрешить внешние запросы к API платформы

## 6. Интерфейс: язык и тема

### Переключение языка

В правой части навигационной панели (и на страницах входа/регистрации) находится кнопка с иконкой языка.

Доступные языки:
- **English**
- **Русский**
- **Қазақша**

Выбранный язык сохраняется в браузере и применяется при следующих посещениях.

### Тёмная / светлая тема

Рядом с переключателем языка — кнопка с иконкой луны (тёмная тема) или солнца (светлая тема). Нажмите для переключения.

## Типичный сценарий работы

1. **Войдите** в платформу
2. Откройте **Карту**, выберите район и время суток
3. Включите **ML рекомендации** — на карте появятся зелёные метки
4. Кликните на интересующую точку — изучите её характеристики
5. Нажмите **Сохранить** на понравившихся точках
6. Перейдите в **Анализ → Рекомендации** — изучите топ предложений с детальным обоснованием
7. Сохраните нужные рекомендации кнопкой **Сохранить**
8. Перейдите в **Анализ → Сохранённые** — нажмите **Открыть полное сравнение**
9. В **Сравнении** переключитесь на «Мои точки» — изучите итоговую таблицу и выберите лучшую локацию
10. Нажмите **Скопировать** в карточке рекомендации — вставьте резюме в отчёт или письмо
