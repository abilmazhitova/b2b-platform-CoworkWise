# Доступ администратора

## Как устроено

- **Админ не регистрируется** через форму на сайте — у него уже есть учётная запись, созданная вручную.
- **Обычный пользователь** регистрируется через «Register», логинится; у него `is_admin = false`, раздел Admin на фронте можно скрывать или ограничивать по роли.
- **Выдать админку другому человеку** можно двумя способами.

---

## 1. Вручную в БД (первый админ или разовый доступ)

Если у тебя есть доступ к PostgreSQL (DBeaver, psql и т.п.):

1. Пользователь уже должен быть в таблице `users` (зарегистрировался через сайт или ты создала запись вручную).
2. Найди его по `email` и поставь `is_admin = true`:

```sql
UPDATE users SET is_admin = true WHERE email = 'email@example.com';
```

Чтобы **создать первого админа с нуля** (пароль нужно захешировать, например через Python):

```bash
cd b2b-backend
.venv\Scripts\python -c "
from app.core.security import hash_password
print(hash_password('твой_пароль'))
"
```

Скопируй хеш и вставь в БД:

```sql
INSERT INTO users (email, hashed_password, full_name, is_active, is_admin)
VALUES ('admin@mail.ru', '$2b$12$...скопированный_хеш...', 'Admin', true, true);
```

---

## 2. Через API (удобно, когда уже есть хотя бы один админ)

Залогинься под учёткой, у которой уже `is_admin = true`, и вызови:

**Список пользователей (только для админа):**
```http
GET /users
Authorization: Bearer <токен_админа>
```

**Выдать/забрать права администратора:**
```http
PATCH /users/{user_id}/admin
Authorization: Bearer <токен_админа>
Content-Type: application/json

{"is_admin": true}
```

- `user_id` — id пользователя из `GET /users` (или из БД).
- Снять админку у **самого себя** нельзя (бэкенд вернёт 400).

Дальше на фронте в разделе Admin → Users можно подставить вызов `GET /users` и кнопку «Сделать админом» с вызовом `PATCH /users/{id}/admin`.

---

## Итого

| Задача | Где |
|--------|-----|
| Создать первого админа | Вручную в БД: `UPDATE users SET is_admin = true` для существующего пользователя или `INSERT` с хешем пароля. |
| Выдать админку ещё кому-то | Либо в БД (`UPDATE ... is_admin = true`), либо через `PATCH /users/{id}/admin` (логин под текущим админом). |
